## BACKEND API ENDPOINTS REQUIRED

GET /api/partner/dashboard
- Returns: visibility score, analytics summary, recent leads

GET /api/partner/leads?status=PENDING
- Returns: filtered lead list with pagination

POST /api/partner/leads/:leadId/status
- Body: { newStatus: 'CONTACTED' | 'CLOSED' | 'REJECTED' }
- Returns: updated lead object

PUT /api/partner/bid
- Body: { category, zipCodes, bidAmount }
- Returns: new visibility score and estimated impressions

GET /api/partner/analytics?range=30d
- Returns: time-series data for chart visualization

## SECURITY CONSIDERATIONS
- Partner can only see their own leads (no cross-partner visibility)
- Commission data encrypted in transit and at rest
- Rate limiting on bid updates (max 5 changes per hour)
- Audit log for all status changes

## MOBILE RESPONSIVENESS
- Stack columns vertically on <768px screens
- Visibility meter converts to vertical progress bar
- Lead table becomes swipeable cards on mobile
- Touch-friendly bid slider with haptic feedback

## NOTIFICATIONS & ALERTS
- Email when new lead arrives (if opted-in)
- SMS for high-value leads (>$10k claim)
- In-app notification when bid falls below competitive threshold
- Weekly digest: "You had 14 new leads this week, 3 closed"
